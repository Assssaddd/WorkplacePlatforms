Computer Network Concepts

1. What is Computer Network?

A computer network is a system of interconnected devices (computers, servers, routers, etc.) that can communicate with each other and share resources like files, printers, and internet connections. Networks can be local (Local Area Networks - LAN) or wide (Wide Area Networks - WAN), and can operate through wired or wireless technologies.

2. Difference Between Router and Switch

  • Router: A router is a device that connects multiple networks (e.g., LAN to WAN) and directs data packets between them. It operates at Layer 3 (Network layer) of the OSI model and makes forwarding decisions based on IP addresses.
  • Switch: A switch is a device that connects devices within a single network (LAN) and forwards data based on MAC addresses. It operates at Layer 2 (Data Link layer) of the OSI model.

3. What is a Broadcast Domain?

A broadcast domain is a logical division of a computer network in which all devices can receive broadcast messages from any other device in the same domain. Devices in different broadcast domains, such as those separated by routers, cannot directly receive each other's broadcast messages.

4. OSI Model with Complete Details and Protocols on Each Layer

OSI Model Layers:

  1. Physical Layer: Deals with physical transmission of data (cables, signals). Protocols: Ethernet, DSL, USB.
  2. Data Link Layer: Manages node-to-node data transfer and error correction. Protocols: Ethernet, PPP, MAC addresses.
  3. Network Layer: Handles routing and addressing. Protocols: IP, ICMP, ARP.
  4. Transport Layer: Ensures reliable data transfer between devices. Protocols: TCP, UDP.
  5. Session Layer: Manages sessions between applications. Protocols: NetBIOS, RPC.
  6. Presentation Layer: Translates data formats for the application. Protocols: JPEG, ASCII, TLS/SSL.
  7. Application Layer: Provides network services to end-user applications. Protocols: HTTP, FTP, DNS, SMTP.

5. DHCP DORA Process

The DORA process is the sequence of events in Dynamic Host Configuration Protocol (DHCP) where:

  • D - Discover: The client sends a DHCPDISCOVER message to locate a DHCP server.
  • O - Offer: The DHCP server responds with a DHCPOFFER message that includes an IP address.
  • R - Request: The client sends a DHCPREQUEST message to request the offered IP address.
  • A - Acknowledge: The server sends a DHCPACK message, confirming the lease of the IP address.

6. DNS â€“ Detailed Explanation

DNS (Domain Name System) is a hierarchical system that translates human-readable domain names (e.g., www.google.com) into IP addresses (e.g., 172.217.14.196). It consists of various components such as DNS servers, zones, and records. The DNS query process typically involves a recursive resolver and authoritative DNS servers that work together to resolve domain names into IP addresses.

7. TCP/UDP and Why?

  • TCP (Transmission Control Protocol): A connection-oriented protocol that ensures reliable, ordered, and error-checked delivery of data. It is used for applications that require high reliability (e.g., HTTP, FTP, email).
  • UDP (User Datagram Protocol): A connectionless protocol that allows fast, best-effort data transfer without guarantees of reliability or order. It is used in applications where speed is critical (e.g., video streaming, VoIP).

8. Subnetting

Subnetting is the practice of dividing a large IP network into smaller, more manageable subnets. This improves network efficiency and security. By modifying the subnet mask, a network administrator can define the size of subnets and allocate IP addresses appropriately.

9. MSS/MTU

  • MTU (Maximum Transmission Unit): The largest size of a data packet that can be transmitted over a network.
  • MSS (Maximum Segment Size): The largest amount of data that can be sent in a single TCP segment, excluding headers. MSS is usually determined by the MTU of the network minus the TCP and IP headers.

10. Complete Flow When You Trigger amazon.com

When you type "amazon.com" into your browser, the following flow occurs:

  • The browser sends a DNS query to resolve the domain name into an IP address.
  • The browser opens a TCP connection to the resolved IP address using port 80 (HTTP) or 443 (HTTPS).
  • The browser sends an HTTP/HTTPS request to the web server (Amazon's server).
  • The server processes the request and sends back the requested webpage data (HTML, CSS, JavaScript).
  • The browser renders the page, displaying it to the user.

11. TCP and SSL Handshake

The TCP handshake is the process of establishing a connection between the client and the server:

  • Step 1: SYN: The client sends a SYN (synchronize) packet to the server to initiate the connection.
  • Step 2: SYN-ACK: The server responds with a SYN-ACK (synchronize-acknowledgment) packet.
  • Step 3: ACK: The client sends an ACK (acknowledgment) packet, completing the handshake.

The SSL handshake occurs after the TCP handshake and is responsible for securing the communication by establishing encryption parameters:

  • The client sends a "ClientHello" message, proposing encryption algorithms.
  • The server responds with a "ServerHello" message and sends its SSL certificate for verification.
  • The client verifies the certificate and generates a pre-master secret to establish a secure session.

12. Flow/Error Control

Flow control ensures that data is transmitted at a rate that the receiver can handle without overwhelming it. It typically uses sliding window techniques in protocols like TCP.

Error control ensures that errors in transmission are detected and corrected. Techniques like checksums, retransmissions, and acknowledgment (ACK) are used for error control in protocols like TCP.

13. What is a Firewall, Why Do You Need It?

A firewall is a network security system that monitors and controls incoming and outgoing traffic based on predefined security rules. It helps prevent unauthorized access to or from private networks and is essential for protecting against cyberattacks, malware, and unauthorized intrusions.

14. What is APIPA?

APIPA (Automatic Private IP Addressing) is a feature in Windows that automatically assigns a private IP address (usually in the range of 169.254.x.x) to a device when it cannot obtain an IP address from a DHCP server.

15. Will My Computer Get the Same IP Address Allocated Every Time?

If you are using DHCP (Dynamic Host Configuration Protocol), your computer may receive a different IP address each time it connects to the network, depending on the DHCP lease. However, if the device is assigned a static IP, it will always get the same IP address.

16. DNS Query Process

The DNS query process occurs when you enter a domain name (e.g., www.amazon.com) into your browser. Here's a breakdown of the process:

  • Step 1: The browser checks the local DNS cache to see if it already knows the IP address for the domain.
  • Step 2: If not found, the browser sends a DNS query to the DNS resolver (usually provided by your ISP).
  • Step 3: The resolver checks if it has the IP address cached; if not, it forwards the query to a root DNS server.
  • Step 4: The root DNS server responds with the IP address of a Top-Level Domain (TLD) DNS server (e.g., .com).
  • Step 5: The resolver sends the query to the TLD server, which responds with the authoritative DNS server for the domain.
  • Step 6: The resolver then queries the authoritative DNS server for the domain (e.g., amazon.com), and the server responds with the IP address of the requested domain.
  • Step 7: The resolver sends the IP address back to the browser, which then uses it to make a connection to the web server and load the page.

17. What Happens if We Type www.amazon.com?

When you type www.amazon.com in the browser, the following series of events occurs:

  • DNS Lookup: The browser initiates the DNS query process to resolve the domain name to an IP address, as explained earlier.
  • TCP Connection: Once the IP address is resolved, the browser initiates a TCP handshake to establish a connection to the Amazon web server.
  • HTTP Request: The browser sends an HTTP request to the server to fetch the webpage.
  • SSL/TLS Handshake: If HTTPS is used, an SSL/TLS handshake is performed to secure the communication between the client and server.
  • Web Page Response: The web server processes the request and sends back the necessary HTML, CSS, JavaScript, and images to the browser.
  • Rendering: The browser renders the page for the user to view.

18. What Are Ports and Port Numbers?

Ports are logical endpoints in a network where communication occurs between two devices. They are used by protocols like TCP and UDP to identify specific services or applications running on a device. Each port is identified by a port number, which is a 16-bit value (ranging from 0 to 65535). Some common well-known port numbers include:

  • 80 - HTTP
  • 443 - HTTPS
  • 25 - SMTP
  • 53 - DNS

19. Do Different Tabs Making Queries in the Browser Use Different Port Numbers?

No, all tabs in a browser typically use the same port number for communication, which is usually port 80 for HTTP or port 443 for HTTPS. However, they may be differentiated by different IP addresses or connections to the same web server with different resources. Each new request made by the browser to the same domain uses the same port for communication, but different sessions can be handled with separate connections.

20. TCP Handshake in Detail

The TCP handshake (also called the three-way handshake) is the process used to establish a connection between a client and a server:

  1. Step 1: SYN - The client sends a SYN (synchronize) message to the server with a sequence number, indicating its desire to establish a connection.
  2. Step 2: SYN-ACK - The server responds with a SYN-ACK message, acknowledging the client's SYN request and sending its own sequence number.
  3. Step 3: ACK - The client sends an ACK message to the server, acknowledging the server's SYN-ACK message. This completes the handshake, and data transmission can begin.

21. How Do PING and TRACERT Commands Work?

Ping is a network diagnostic tool used to check the availability and round-trip time of a remote host by sending ICMP Echo Request packets and receiving ICMP Echo Replies. The time it takes for the request to travel to the host and back is displayed.

Traceroute (tracert) traces the route that packets take to reach a destination host. It sends multiple ICMP Echo Request packets with incrementing TTL (Time to Live) values, recording the IP addresses of each router along the path until the destination is reached.

22. Problem on IP Addressing at the Sender, Gateway, and Destination

If there is an IP addressing issue, the flow of data between sender, gateway, and destination will be interrupted. Common problems include:

  • Incorrect IP address: If the sender or destination has an incorrect IP address, the packet cannot be routed to the correct host.
  • Incorrect Gateway: If the sender's default gateway is incorrect, it won't know how to forward packets to the destination outside of its local network.
  • Routing Issues: If there are misconfigured routes or unreachable gateways, data cannot flow to the destination.

23. Detail Discussion on IPv4 and IPv6

  • IPv4: IPv4 (Internet Protocol version 4) is the fourth version of the IP protocol and uses a 32-bit address format (e.g., 192.168.1.1). IPv4 supports around 4.3 billion unique IP addresses.
  • IPv6: IPv6 (Internet Protocol version 6) is the successor to IPv4 and uses a 128-bit address format (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334), supporting a vastly larger number of unique addresses. IPv6 also improves upon IPv4 by providing features like auto-configuration and better security.

24. How Do We Check IP Address, Gateway, and DNS Server Addresses of Our Device?

On most devices, you can check your IP address, default gateway, and DNS server addresses using the following commands:

  • Windows: Use the ipconfig command in Command Prompt.
  • Linux/MacOS: Use the ifconfig (or ip addr show) command in the terminal.

25. How Does HTTPS Secure Our Connection?

HTTPS (Hypertext Transfer Protocol Secure) uses SSL/TLS protocols to encrypt communication between the client and server, ensuring that sensitive data (e.g., passwords, credit card information) is transmitted securely. It also verifies the identity of the server, preventing man-in-the-middle attacks.

26. Know About Header (TCP, IP, etc.)

Headers are sections of metadata in data packets that provide important information about the packet, such as source and destination addresses, protocol used, and sequence numbers:

  • IP Header: Contains the source and destination IP addresses, version of IP (IPv4 or IPv6), TTL, and more.
  • TCP Header: Includes source and destination port numbers, sequence and acknowledgment numbers, flags (SYN, ACK), window size, and checksum.

27. Linux Command (ping, traceroute, dig, curl, mpstat, etc.)

  • ping: Sends ICMP Echo Requests to check connectivity to a remote host and measure round-trip time.
  • traceroute: Traces the path of packets to a destination host, showing all intermediate routers along the way.
  • dig: A DNS query tool used to retrieve information from DNS servers, including IP addresses and other records for a domain.
  • curl: A command-line tool used to transfer data from or to a server using various protocols (HTTP, FTP, etc.). It is commonly used to test HTTP responses and interact with REST APIs.
  • mpstat: A tool used to display CPU utilization statistics for each processor (CPU) on a system. It helps in monitoring performance and diagnosing bottlenecks.

28. What is SSL? Explain About Digital Certificates and TLS Handshake.

SSL (Secure Sockets Layer) is a cryptographic protocol designed to provide secure communication over a computer network. It encrypts the data transferred between a web server and a browser to protect sensitive information such as login credentials and credit card numbers.

Digital Certificates are used to prove the authenticity of a website or a server during an SSL/TLS handshake. They are issued by trusted Certificate Authorities (CAs) and contain the serverâ€™s public key and information about its identity.

TLS Handshake is the process that establishes a secure communication channel between the client and the server. It involves the following steps:

  • The client sends a "ClientHello" message, proposing encryption algorithms.
  • The server responds with a "ServerHello" message, choosing the encryption method and sending its SSL certificate.
  • The client verifies the server's certificate and generates a pre-master secret.
  • Both the client and the server generate session keys from the pre-master secret, which are used to encrypt subsequent communication.
  • The secure communication channel is now established.

29. What Are a Few Methods of HTTP, What is HTTPS, Is the Port Number Different for HTTPS, How Is SSL Related to HTTPS?

HTTP Methods are standard request types used in HTTP communication, including:

  • GET: Retrieves data from the server.
  • POST: Sends data to the server to create or update a resource.
  • PUT: Replaces an existing resource on the server with the provided data.
  • DELETE: Deletes a specified resource from the server.

HTTPS (Hypertext Transfer Protocol Secure) is the secure version of HTTP, where all communication is encrypted using SSL/TLS protocols. The primary difference from HTTP is that HTTPS ensures secure data transmission over the network, preventing eavesdropping and tampering.

Port Number: By default, HTTP operates on port 80, while HTTPS operates on port 443.

SSL and HTTPS: SSL (or TLS) is the protocol used to secure HTTP traffic. HTTPS relies on SSL/TLS to encrypt data, ensuring that sensitive information remains confidential during transmission.

30. What Methods Are Available to Us in Flow Control and Error Detection/Correction?

Flow Control methods are used to prevent network congestion and ensure that the sender does not overwhelm the receiver. Some common flow control techniques include:

  • Window-based flow control: Used by protocols like TCP, where the sender can send multiple packets up to the size of the "window" without waiting for an acknowledgment.
  • Credit-based flow control: The receiver grants the sender permission to transmit data based on available buffer space.

Error Detection/Correction ensures that transmitted data is accurate. Common methods include:

  • Checksums: Used to detect errors in transmitted data by comparing the checksum value calculated at the sender and receiver.
  • Parity Bits: Adds an extra bit to data to detect errors in transmission.
  • Cyclic Redundancy Check (CRC): A method used for error detection, commonly used in Ethernet frames.
  • Retransmission: A process where lost or corrupted data packets are retransmitted until successful delivery is confirmed.

31. What Is Public IP? How Is It Different from Private IP? Can I Connect to the Internet Using Private IP? What Is NAT?

  • Public IP: A public IP address is a unique address assigned to a device on the internet, allowing it to be directly reachable from anywhere on the network.
  • Private IP: A private IP address is used within a local network and is not routable over the internet. Private IPs are defined by specific ranges (e.g., 192.168.x.x, 10.x.x.x).
  • Can I Connect to the Internet Using Private IP? No, private IPs cannot communicate directly with the internet. However, devices using private IPs can access the internet through Network Address Translation (NAT).
  • NAT: NAT (Network Address Translation) is a method used by routers to allow multiple devices with private IPs to share a single public IP address when accessing the internet. It translates private IP addresses to a public IP address for outgoing traffic and vice versa for incoming traffic.

32. System Automation â€“ Cron, Batch Jobs, Windows Startup Tasks

Cron is a time-based job scheduler in Unix-like systems that allows scheduled tasks (or cron jobs) to be executed at specified intervals. Common uses include running backup scripts, sending emails, or cleaning temporary files.

Batch Jobs are scripts or programs that can be executed automatically on a set schedule in Windows. They are often used for system maintenance or automating repetitive tasks.

Windows Startup Tasks are programs or scripts configured to run automatically when Windows starts. These tasks can be configured via the Task Scheduler or placed in the Startup folder.

33. Firewall in Detail with All Types (Advantages/Disadvantages)

A firewall is a network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules. It is used to protect networks and devices from unauthorized access and cyberattacks.

Types of Firewalls:

  • Packet Filtering Firewalls: Inspect packets at the network layer and allow or block traffic based on predefined rules.
  • Stateful Inspection Firewalls: Track the state of active connections and make decisions based on the state of the connection.
  • Proxy Firewalls: Act as intermediaries between users and services, filtering traffic and performing deep packet inspection.
  • Next-Generation Firewalls (NGFW): Combine traditional firewall features with advanced capabilities like intrusion prevention and application awareness.

Advantages:

  • Helps protect against unauthorized access and cyberattacks.
  • Can block harmful content, malware, and viruses.
  • Provides network monitoring and logging for auditing purposes.

Disadvantages:

  • Can introduce latency or performance issues if not configured correctly.
  • May block legitimate traffic if overly restrictive rules are applied.

34. Differentiate OSI Reference Model with TCP/IP Reference Model

OSI Model:

  • It has 7 layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.
  • Designed by ISO (International Organization for Standardization) to standardize network protocols.
  • Each layer is responsible for specific networking tasks, such as physical transmission, routing, and data formatting.
  • It is a theoretical model used for understanding and designing network systems.

TCP/IP Model:

  • The TCP/IP model has 4 layers: Link, Internet, Transport, and Application.
  • It was developed by the Department of Defense (DoD) for the ARPANET (precursor to the internet).
  • It is more practical and widely used compared to the OSI model, especially in the design and implementation of networking protocols.
  • The TCP/IP model focuses on the internet protocols and their application for communication over the internet.

Comparison:

Layer in OSI Model Layer in TCP/IP Model Responsibilities
Physical Link Deals with hardware transmission and physical media (cables, switches).
Data Link Link Defines how data is formatted and transmitted across the network.
Network Internet Responsible for routing packets between different networks (IP addressing).
Transport Transport Ensures end-to-end communication (TCP/UDP protocols, flow control, error recovery).
Session Application Manages sessions between applications (establishes, maintains, and terminates connections).
Presentation Application Handles data translation, encryption, and compression (e.g., SSL, file formats).
Application Application Deals with high-level protocols like HTTP, FTP, DNS, etc. that facilitate user interaction.

Summary of Differences:

  • The OSI model is a more detailed, theoretical framework with seven layers, while the TCP/IP model is simpler with four layers.
  • The OSI model includes layers for session management and data presentation, which are combined into the application layer in TCP/IP.
  • The OSI model is used for designing and understanding networks, whereas the TCP/IP model is more practical and widely implemented in real-world networking.
